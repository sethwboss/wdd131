Rock Paper Scissors Site Plan

Seth Boss

WDD 131-3

Overview

Purpose

To provide users with a fun game experience against the computers

Audience

Anyone computer-oriented or who likes video games

Dynamic elements

The game interface on the child page is dynamically loaded, and then alters after the user selects the picture for 'rock', 'paper', or 'scissors. After the input, the computer's selection appears on the screen. The input is then evaluated and measured against the computer's guess, and a popup is loaded based on whether the user won or lost. A counter detailing how many times the user has won, lost, or tied is updated after each round.

Wireframes

Home

The Game link in the header will take them to the game page, but the contact link will scroll them down to the footer, where contact info is available for questions.

Game Page

The box around the computer's selection represents a covering, that will be shown before the user submits. After the user submits their selection, the box will be changed to hidden to show what the computer selected. Then, it will be hidden again once the user clicks on one of their pictures for the new round.
